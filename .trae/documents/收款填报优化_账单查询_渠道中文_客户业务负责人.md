# 收款填报优化 · 账单查询 · 渠道中文化 · 客户业务负责人

## 一、Context（背景）

用户提出 5 项优化需求，针对收款填报/核对、账单管理、客户管理的体验问题：

1. **收款填报不需要区分款项性质**：当前 PaymentSubmit.vue 有"款项性质"字段（公款/私款），但后端 `create_payment` 已强制 `usage_type="public"`，前端字段无效且冗余。
2. **选择截图按钮样式奇怪**：当前 `<el-button type="primary">选择截图</el-button>` 与"提交收款"按钮样式完全相同，视觉上混淆。
3. **账单管理需要查询功能**：当前 BillList.vue 仅有客户/状态/年/月筛选，缺"跟进人"筛选；且后端 `_serialize_bill` 未返回 `follow_up_user_name`，导致前端"跟进人"列始终为空。
4. **收款核对中渠道显示英文**：PaymentVerify.vue L17 直接显示 `row.channel`（如 wechat/bank），应显示中文。
5. **客户管理增加业务负责人**：客户当前只有"销售负责人"(sales_person_id)和"介绍人"(introducer_user_id)，缺"业务负责人"字段。账单生成时应把客户当时的业务负责人快照到账单的"跟进人"字段（经用户确认，业务负责人=跟进人，合并为一个字段）。

### 需求5：客户管理增加业务负责人字段

**`backend/app/models/__init__.py`**
- Customer(Company) 模型增加字段：
  ```python
  business_owner_id = Column(Integer, ForeignKey("users.id"), nullable=True)
  ```
  外键关联 users.id，可空

**`backend/app/routers/customers.py`**
- `create_customer` / `update_customer` 接受 `business_owner_id` 字段
- `list_customers` / `get_customer` 返回 `business_owner_id` 和 `business_owner_name`（关联查询 User.name）

**`backend/app/routers/bills.py`**
- `generate_bills`（L121-157）修改逻辑：
  - 生成账单时，先查 customer.business_owner_id
  - 若 customer.business_owner_id 不为空 → `bill.follow_up_user_id = customer.business_owner_id`
  - 若为空 → fallback 到 `sub.service_owner_id`（保留现有逻辑）
  - 这样既满足"账单填客户业务负责人"，又保持向后兼容

**`frontend/src/views/CustomerList.vue`**
- 表格增加"业务负责人"列（在"销售负责人"后）
- 编辑/新增弹窗增加"业务负责人"下拉（options 来自 users，与"销售负责人"一致）
- form 增加 `business_owner_id` 字段

**`frontend/src/views/BillList.vue`**
- 表格"跟进人"列（L33）改名为"业务负责人"（与需求5统一概念）

### 数据库迁移

Customer 表新增 `business_owner_id` 字段。由于项目使用 SQLAlchemy `Base.metadata.create_all`，开发环境重启后端会自动建表（create_all 不修改已有表）。需要手动执行 ALTER TABLE：
```sql
ALTER TABLE companies ADD COLUMN business_owner_id INT NULL;
```
或通过后端启动时执行迁移脚本。最简方案：在 `app/main.py` startup 事件中执行 raw SQL 检查并添加字段（参考现有 startup 模式）。

## 三、Assumptions & Decisions

1. **需求1**：后端 `create_payment` 已强制 `usage_type="public"`，前端删除字段不影响后端。PaymentVerify 的"性质"列同步删除（已无意义）。
2. **需求2**：用 `text` 文字按钮 + Upload 图标，最轻量，与"提交收款"主按钮视觉区分明显。
3. **需求3**：后端 `_serialize_bill` 当前未返回 `follow_up_user_name`，这是已有 bug（前端表格该列始终为空），本次顺带修复。
4. **需求4**：渠道映射规则：bank→银行转账、alipay→支付宝、wechat→微信、cash→现金。PaymentSubmit.vue 的渠道下拉已是中文 label，无需改动。
5. **需求5**：用户确认"业务负责人"="跟进人"，合并为一个字段。Customer 新增 `business_owner_id`，账单生成时填到现有 `Bill.follow_up_user_id`（不新增 Bill 字段）。账单"跟进人"列改名为"业务负责人"。
6. **fallback 逻辑**：客户未设置业务负责人时，账单 follow_up_user_id 仍用 sub.service_owner_id（保持向后兼容，不破坏现有月结计算）。
7. **不修改 conftest.py 和现有测试**：新增字段可空，不影响现有测试。可选新增 1-2 个测试验证 business_owner_id 功能。
8. **前端颜色不引入蓝紫色调**：保持现有米色/棕色系。
