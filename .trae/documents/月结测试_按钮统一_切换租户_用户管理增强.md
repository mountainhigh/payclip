### 需求 4：用户管理增强

#### 2.7 `backend/app/routers/users.py`
- **`list_users`** 改造：
  ```python
  # super_admin 切换租户后：返回选中租户用户 + 自己
  if user.role == "super_admin":
      if user.tenant_id is not None:
          # 已切换租户
          q = db.query(User).filter((User.tenant_id == user.tenant_id) | (User.id == user.id))
      else:
          # 未切换：返回全部
          q = db.query(User)
  else:
      q = db.query(User).filter(User.tenant_id == user.tenant_id)
  ```
  - 返回字段增加 `"tenant_id": u.tenant_id`

- **`update_user`** 改造：增加 `role` 字段支持
  ```python
  if "role" in body:
      new_role = body["role"]
      if new_role not in ("super_admin", "tenant_admin", "employee"):
          raise HTTPException(400, "无效的角色")
      # tenant_admin 只能在 employee/tenant_admin 间切换
      if user.role == "tenant_admin" and new_role not in ("employee", "tenant_admin"):
          raise HTTPException(403, "租户管理员只能设置员工或租户管理员角色")
      # tenant_admin 不能改 super_admin 角色的用户
      if user.role == "tenant_admin" and u.role == "super_admin":
          raise HTTPException(403, "无权修改平台管理员")
      # 跨租户校验：super_admin 切换后只能改选中租户用户或自己
      if user.role == "super_admin" and u.tenant_id != user.tenant_id and u.id != user.id:
          raise HTTPException(403, "无权操作其他租户用户")
      u.role = new_role
  ```

## 三、Assumptions & Decisions

1. **统一样式保留功能**：SubscriptionList（3 按钮）/OnetimeList（2 按钮）保留各自功能按钮，只套用 CustomerList 的样式（小尺寸 + flex 包裹 + 紧凑 padding）。SubscriptionList 列宽从 220→180（3 按钮需更宽），OnetimeList 从 200→120（2 按钮足够）。
2. **super_admin 切换租户后读写均生效**：通过 X-Tenant-Id header 临时覆盖 `user.tenant_id`，所有按 tenant_id 过滤的接口自动生效；后端 admin.py 接口（租户管理/注册码管理）天然不受影响（用 `require_super_admin`，无 tenant 过滤）。
3. **tenant_admin 可改员工角色**：employee↔tenant_admin 间切换；super_admin 可设任意角色。
4. **切换租户后强制刷新页面**：用 `window.location.reload()` 确保所有页面数据重新拉取（最简单可靠）。
5. **actingTenantId 持久化到 localStorage**：刷新页面后保持选择。
6. **登录默认选第一个租户**：super_admin 登录后若 localStorage 无 actingTenantId，自动选租户列表第一个。
7. **不修改 conftest.py**：月结测试数据在 test_ledger.py 内本地构造，避免影响其他测试。
8. **前端颜色不引入蓝紫色调**：维持现有米色/棕色系，el-select 用默认样式但通过 CSS 变量适配（如有必要）。
