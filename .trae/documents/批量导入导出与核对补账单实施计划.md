# 批量导入导出 + 收款核对补账单 实施计划

## Context（背景与目标）

用户在试用系统时发现两类问题：

1. **数据录入效率低**：客户、供应商、长期业务、一次性业务、账单都需要逐条手工录入，无法批量导入。当用户从旧系统迁移数据或需要导出对账时没有工具支持。
2. **收款核对流程不完整**：收款填报时虽然支持选客户后列出未付账单（[PaymentSubmit.vue:99-103](file:///e:/code/ai_code/payclip/frontend/src/views/PaymentSubmit.vue#L99-L103) 已实现），但如果填报时没选账单（直接提交），核对阶段（[PaymentVerify.vue](file:///e:/code/ai_code/payclip/frontend/src/views/PaymentVerify.vue)）只能通过/驳回，**无法补充账单分配**，导致账单状态无法更新。

本计划为 5 个列表页增加 Excel 批量导入/导出，并为收款核对增加"补充账单分配"功能。

**已确认的设计决策**（来自用户）：
- 账单导入用于「迁移历史账单」（直接写入，绕过自动生成）
- 导入冲突策略：按名称匹配，已存在则覆盖更新
- 关联字段用名称填写（如客户名、员工名），后端自动查 ID
- 核对补账单用「弹窗编辑分配」交互

**附带**：[OnetimeList.vue:11](file:///e:/code/ai_code/payclip/frontend/src/views/OnetimeList.vue#L11) 的列标签当前已是"业务类型"（无需修改）。

### 二、后端：5 个模块各加导入/导出接口

每个模块的 router 增加两个接口，路由前缀沿用现有：

| 模块 | 导出接口 | 导入接口 |
|---|---|---|
| 客户 | `GET /api/customers/export` | `POST /api/customers/import` |
| 供应商 | `GET /api/suppliers/export` | `POST /api/suppliers/import` |
| 长期业务 | `GET /api/subscriptions/export` | `POST /api/subscriptions/import` |
| 一次性业务 | `GET /api/onetime-projects/export` | `POST /api/onetime-projects/import` |
| 账单 | `GET /api/bills/export` | `POST /api/bills/import` |

**导出实现**（统一模式）：
- 接收与 list 接口相同的筛选参数（keyword/company_id/year/month 等）
- 查询当前租户的全部匹配数据（不分页）
- 调用 `export_rows()` 返回 `StreamingResponse`，Content-Type=`application/vnd.openxmlformats-officedocument.spreadsheetml.sheet`，带 `Content-Disposition: attachment; filename=xxx.xlsx`

**导入实现**（统一模式）：
- 接收 `UploadFile`，读取字节后调 `parse_import_file()`
- 对每行：
  1. 用名称解析关联 ID（如 `customer_name → company_id`），调 `db.query(Company).filter(Company.name==name, tenant_id==...).first()`，找不到则该行加入 errors
  2. 按唯一键（客户/供应商按 name；业务按 company_id+service_type/project_type；账单按 company_id+bill_type+billing_year+billing_month）查现有记录：存在则更新，不存在则新建
  3. 已锁定月份的账单不允许导入（同 [payments.py:85-91](file:///e:/code/ai_code/payclip/backend/app/routers/payments.py#L85-L91) 的锁定校验）
- 返回 `{success_count, updated_count, created_count, errors: [{row, reason}]}`

**字段映射**（Excel 表头 → 后端字段）：

```
客户：客户名称, 状态, 新客户(是/否), 合作起始, 区域标签(分号分隔), 介绍人类型, 介绍人名称, 销售负责人, 联系电话, 邮箱, 备注
供应商：供应商名称, 类型, 联系人, 备注
长期业务：客户名称, 业务类型, 计费周期, 月费, 是否成本类型, 月成本, 供应商, 服务负责人, 销售负责人, 起始日期
一次性业务：客户名称, 业务类型, 收入, 成本, 供应商, 负责人, 完成日
账单：客户名称, 账单类型, 账单年份, 账单月份, 应收金额, 已收金额, 付款状态, 备注
```

**权限**：导出需 `get_current_user`（任何租户用户）；导入需 `require_write_access` 或 `tenant_admin`（同各模块的 create 接口）。

### 三、后端：核对阶段补充账单分配接口

**修改文件**：[backend/app/routers/payments.py](file:///e:/code/ai_code/payclip/backend/app/routers/payments.py)

新增接口 `PUT /api/payments/{pid}/allocations`：
- 权限：`payment:verify` 或 `tenant_admin`
- 请求体：`{ bill_allocations: [{ bill_id, allocation_amount }] }`
- 逻辑：
  1. 校验收款存在且属于当前租户
  2. 校验该月未锁定（同 create_payment 的锁定校验）
  3. 删除该 payment 现有的 `PaymentBillAllocation`（仅未核对的；已核对的回收原账单 paid_amount 后重置）
  4. 写入新的分配记录
  5. 重新计算客户预付款（同 create_payment 的预付款逻辑）
- 返回 `{ message: "分配已更新" }`

### 五、前端：通用导入导出组件

**新增文件**：[frontend/src/components/ImportExportButtons.vue](file:///e:/code/ai_code/payclip/frontend/src/components/ImportExportButtons.vue)

Props：
- `exportUrl`：导出接口路径（如 `/customers/export`）
- `importUrl`：导入接口路径
- `exportParams`：导出时的筛选参数对象
- `templateUrl`：模板下载路径（可选）

功能：
- 「导出 Excel」按钮：调 `request.get(url, { params, responseType: 'blob' })`，用 `URL.createObjectURL` 触发下载
- 「导入 Excel」按钮：`el-upload` 选文件后调 `request.post(url, formData, multipart)`，返回结果用 `ElMessageBox.alert` 显示「成功 X 条，更新 Y 条，新建 Z 条，失败 N 条」+ 错误明细
- 「下载模板」按钮：调 `/api/{module}/template` 下载空白模板（后端各模块加一个 template 接口返回只含表头的 xlsx）
