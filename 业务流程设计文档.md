```mermaid
flowchart TD
```

### 1.3 核对流程

```mermaid
flowchart TD
```

### 1.4 提成计算引擎详细流程

```mermaid
flowchart TD
    START["ledger模块触发<br/>calculate(year, month)"] --> LOCK["锁定当月收款<br/>(禁止新增/修改)"]
    LOCK --> GET_BILLS["获取当月所有Bill<br/>WHERE billing_year=year<br/>AND billing_month=month"]
    GET_BILLS --> ITER["遍历每张Bill"]

    ITER --> TYPE_CHECK{"bill_type?"}

    TYPE_CHECK -- "subscription" --> SUB_LOGIC["长期业务逻辑"]
    TYPE_CHECK -- "onetime" --> OT_LOGIC["一次性业务逻辑"]

    subgraph SUB_LOGIC["长期业务提成"]
        S1["获取Subscription"] --> S2["取有效月费<br/>(FeeHistory按生效日期)")
        S2 --> S3["服务提成 = 月费 × 0.15"]
        S3 --> S4{"Bill.payment_status?"}
        S4 -- "unpaid/overdue" --> S5["欠费扣款 = 月费 × 0.05<br/>创建CommissionDetail<br/>(deduction>0)"]
        S4 -- "paid" --> S6{"有历史扣款?"}
        S6 -- "有" --> S7["补回总额 = 历史所有扣款<br/>创建CommissionDetail<br/>(is_supplement=true)"]
        S6 -- "无" --> S8["正常计提<br/>no deduction"]
        S4 -- "partial" --> S9["正常计提<br/>no deduction<br/>no supplement"]
        S5 --> S10["检查销售提成"]
        S7 --> S10
        S8 --> S10
        S9 --> S10
        S10 --> S11{"新客+12月窗口内?"}
        S11 -- "是" --> S12["销售提成 = 当月收款 × 0.15<br/>创建CommissionDetail(type=sales)"]
        S11 -- "否" --> S13["跳过"]
    end

    subgraph OT_LOGIC["一次性业务提成"]
        O1["获取OneTimeProject"] --> O2["毛利 = revenue - cost"]
        O2 --> O3["一次性提成 = 毛利 × 0.20"]
        O3 --> O4{"is_received?"}
        O4 -- "否" --> O5["欠费扣款 = 毛利 × 0.05"]
        O4 -- "是" --> O6["正常计提<br/>no deduction"]
    end

    S12 --> NEXT
    S13 --> NEXT
    O5 --> NEXT
    O6 --> NEXT
    NEXT["下一条Bill"] -->|"继续"| ITER
    NEXT -->|"完毕"| SUMMARY["按user_id汇总<br/>生成MonthlySalary"]
    SUMMARY --> FINISH["标记计算完成"]
```

### 1.5 补回逻辑详解

```mermaid
flowchart TD
    TRIGGER["Bill.payment_status = paid<br/>(收足当月触发)"] --> FIND["查找该Bill对应Subscription<br/>历史上的所有欠费扣款记录"]
    FIND --> FILTER["过滤: deduction_amount > 0<br/>且未补回过(无对应supplement记录)"]
    FILTER --> CHECK{"有历史扣款?"}
    CHECK -- "有" --> GET_OWNER["通过subscription_owner_histories<br/>查找扣款月份的实际负责人"]
    GET_OWNER --> CALC["补回总额 = SUM(deduction_amount)"]
    CALC --> CREATE["创建CommissionDetail:<br/>user_id = 原被扣人<br/>commission_amount = 0<br/>supplement_amount = 补回总额<br/>is_supplement = true<br/>supplement_for_user_id = 原被扣人"]
    CREATE --> DONE["计入该月应发薪资"]
    CHECK -- "无" --> SKIP["跳过"]
```

## 2. 单据状态机

### 2.1 收款记录状态机

```mermaid
stateDiagram-v2
    [*] --> pending: 员工提交
    
    pending --> approved: 主管核对通过
    pending --> rejected: 主管驳回(二次确认)
    
    rejected --> pending: 员工修改重提
    
    approved --> [*]: public→进入计算
    approved --> [*]: private→排除计算
```

### 2.2 账单状态机

```mermaid
stateDiagram-v2
    [*] --> unpaid: 系统生成账单
    
    unpaid --> partial: 收款分配(paid < receivable)
    unpaid --> paid: 收款分配(paid >= receivable)
    unpaid --> overdue: 计算月到达且仍unpaid
    
    partial --> paid: 补充收款(paid >= receivable)
    partial --> overdue: 计算月到达且仍partial
    
    overdue --> partial: 后续有收款分配
    overdue --> paid: 后续收款补足
    
    paid --> [*]: 收足,触发补回
```

### 2.3 内账校验锁状态机

```mermaid
stateDiagram-v2
    [*] --> unlocked: 月份到达
    
    unlocked --> calculating: 财务点击「校验并计算当月」
    calculating --> locked: 计算完成(自动锁定)
    calculating --> calculating: 防重复(状态锁)
    
    locked --> unlocked: 撤销最近一月锁定
    locked --> [*]: 历史月不可撤销
    
    unlocked --> [*]: 等待操作
```

### 2.4 核对状态转换表

| 当前状态 | 操作 | 目标状态 | 条件 |
|---------|------|---------|------|
| pending | 通过 | approved | 核对人有 verify 权限 + 该收款分配给自己 |
| pending | 驳回 | rejected | 二次确认 |
| rejected | 修改重提 | pending | 提交人可修改 |
| approved | — | (终态) | 不可更改 |

## 3. 月结流程（完整）

```mermaid
sequenceDiagram
    participant 财务 as 财务
    participant 前端 as 前端
    participant API as FastAPI
    participant Ledger as Ledger模块
    participant Payment as Payment模块
    participant Engine as Commission引擎
    participant Salary as Salary模块
    participant DB as MySQL

    Note over 财务,DB: ===== 月结操作 =====
    
    财务->>前端: 点击「校验并计算当月」
    前端->>API: POST /api/ledger/validate-and-calculate
    API->>Ledger: validate_and_calculate(year, month)
    
    Ledger->>DB: 查询本月是否已锁定
    alt 已锁定
        Ledger-->>API: 409 该月已锁定
        API-->>前端: 错误提示
    else 未锁定
        Ledger->>DB: 创建/更新 ledger_validations<br/>状态: calculating
        
        Ledger->>Payment: lock_month(year, month)
        Payment->>DB: 标记当月收款不可编辑
        
        Ledger->>Engine: calculate(year, month)
        Engine->>DB: 读取当月Bill+Payment
        Engine->>Engine: 计算三种提成+欠费+补回
        Engine->>DB: INSERT commission_details
        Engine-->>Ledger: 计算完成
        
        Ledger->>Salary: generate(year, month)
        Salary->>DB: 按user汇总提成
        Salary->>DB: INSERT monthly_salaries
        Salary-->>Ledger: 薪资生成完成
        
        Ledger->>DB: 更新 ledger_validations<br/>状态: locked
        Ledger-->>API: 完成
        API-->>前端: {ledger_id, status:locked, salary_count, commission_count}
        前端-->>财务: 成功提示 + 查看薪资
    end
```

## 4. 异常处理流程

### 4.1 欠款删除拦截

```mermaid
flowchart TD
    DEL["用户点击删除客户/业务"] --> CHECK{"该客户/业务下<br/>是否有未结清Bill?"}
    CHECK -- "有(paid<receivable)" --> REFUSE["422: 禁止删除<br/>提示: 有未收款,请先结清"]
    CHECK -- "无" --> SOFT["软删除: is_archived=1"]
    SOFT --> DONE["操作完成"]
```

### 4.2 预付款自动抵扣流程

```mermaid
flowchart TD
    NEW_BILL["新账单生成"] --> CHECK_PRE{"该客户是否有<br/>预付余额(customer_prepayments.balance>0)?"}
    CHECK_PRE -- "有" --> CALC_USE["计算可抵扣金额<br/>= MIN(balance, receivable_amount)"]
    CALC_USE --> CREATE_ALLOC["创建PaymentBillAllocation<br/>source='prepayment'"]
    CREATE_ALLOC --> UPDATE_BALANCE["customer_prepayments.balance -= 抵扣金额"]
    UPDATE_BALANCE --> UPDATE_BILL2["Bill.paid_amount += 抵扣金额"]
    UPDATE_BILL2 --> UPDATE_STATUS["更新Bill.payment_status"]
    CHECK_PRE -- "无" --> SKIP2["跳过"]
```

## 5. 风险点

| 风险 | 缓解 |
|------|------|
| 月结计算中途失败, 部分数据写入 | 整个计算过程用 DB 事务包裹, 失败回滚 |
| 并发月结触发 (状态锁绕过) | Ledger模块 checking calculating 状态 + DB 唯一约束 |
| 补回归属人查找失败 | subscription_owner_histories 覆盖所有变更, 兜底取当前负责人 |
| 预付款抵扣与收款分配冲突 | 预付款抵扣和手动分配在同一事务中顺序执行 |
