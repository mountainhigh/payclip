## 4. 发现并修复的 BUG

### 4.1 【严重】移动端整站无法加载

**文件**：`mobile/src/App.vue:8`

**原因**：`import { ConfigProviderThemeVars } from 'vant'` —— Vant 没有这个具名导出（它是 TypeScript 类型，不是运行时值），导致 main.js 加载抛 `SyntaxError`，整个 mobile app 不 mount，页面空白。

**修复**：删除该 import。

**验证**：移动端登录页、工作台、收款填报/核对、薪资、个人中心全部正常渲染。

---

### 4.2 【严重】后端 super_admin 创建租户数据 500 错误

**文件**：`backend/app/main.py` 多个 create 接口 + `backend/app/core/auth.py`

**原因**：所有租户级实体（customers/suppliers/subscriptions/onetime/bills/payments/users/cost-presets/bonus-tiers/ledger）的 `tenant_id` 列为 `nullable=False`，但 super_admin 的 `tenant_id=None`。super_admin 调用任何 create 接口都触发 `IntegrityError: Column 'tenant_id' cannot be null`，返回 HTTP 500。

**修复**：在 `core/auth.py` 新增 `require_tenant_user` 依赖（拒绝 super_admin + 检查只读降级），把 12 个 create 类接口的 `require_write_access` 改为 `require_tenant_user`。

**验证**：super_admin 创建租户数据现在返回 403 + `"平台管理员不能直接创建租户数据，请切换到租户账号"`；租户用户正常创建；只读降级状态写操作返回 403 + `"套餐已过期..."`。

---

### 4.3 【中】PC 端驳回收款原因丢失

**文件**：`frontend/src/views/PaymentVerify.vue:72,77`

**原因**：前端传 `reason` 字段，后端读 `reject_reason`（`backend/app/main.py:939`），字段名不一致，驳回原因永远存空。

**修复**：前端两处（单条驳回 + 批量驳回）`reason` → `reject_reason`。

**验证**：API 测试 `actual_reason == "测试驳回原因"` ✅

---

### 4.4 【中】PC 端一次性业务"编辑"按钮无效

**文件**：`frontend/src/views/OnetimeList.vue:78`

**原因**：`function openEdit(row) { /* read-only cost/revenue edit */ }` 函数体为空，点击编辑按钮无任何反应；同时 onSave 只走 create 分支，不支持 update。

**修复**：实现 openEdit（回填表单 + 打开弹窗），onSave 根据 `form.id` 走 create 或 update，弹窗标题动态切换。

**验证**：API 层面 create→update→查询，revenue 10000→12000、cost 3000→3500 均正确写入。

---

### 4.5 【轻】PC 端硬编码颜色与莫兰迪主题不符

**文件**：`frontend/src/views/SalaryList.vue`、`SalaryDetail.vue`、`OnetimeList.vue`、`PaymentSubmit.vue`

**问题**：扣款用 `#f56c6c`、补回/毛利用 `#67c23a`、应发用 `#409eff`（Element Plus 默认蓝），与全局莫兰迪藕粉主题（`--primary: #C68B8B`）冲突。

**修复**：全部改用 CSS 变量 `var(--danger) / var(--success) / var(--primary)`。

---

### 4.6 【轻】PC 端分页器当前页码用默认蓝色

**文件**：`frontend/src/style.css`

**问题**：Element Plus 的 `--el-pagination-active-bg-color` 默认 `#409eff`，原 CSS 只改了 `active-color` 没改背景，分页器当前页是蓝色按钮。

**修复**：补全 6 个分页器 CSS 变量 + 加 `.el-pagination .el-pager li.is-active` 强制覆盖。

**验证**：computed `backgroundColor` 从 `rgb(64,158,255)` 变为 `rgb(198,139,139)` ✅

---

### 4.7 【轻】AdminTenants/AdminRegCodes 缺 page-container

**文件**：`frontend/src/views/AdminTenants.vue`、`AdminRegCodes.vue`

**问题**：根 `<div>` 没加 `page-container` class，与其他页面结构不一致（视觉上因 MainLayout 自带 padding 差异不大，但 DOM 结构不统一）。

**修复**：补 `page-container` + `toolbar` class，标题包进 toolbar。
