## 3. 本地运行

### 3.1 准备 MySQL 8.0

```bash
# 启动本地 MySQL（已安装的前提下）
sudo service mysql start

mysql -e "CREATE DATABASE IF NOT EXISTS salary_manager CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;"
mysql -e "CREATE USER IF NOT EXISTS 'salary'@'localhost' IDENTIFIED BY 'salary123';"
mysql -e "GRANT ALL PRIVILEGES ON salary_manager.* TO 'salary'@'localhost'; FLUSH PRIVILEGES;"
```
默认连接串：`mysql+pymysql://salary:salary123@localhost:3306/salary_manager`
（可用环境变量 `DATABASE_URL` 覆盖。）

### 3.2 启动后端

```bash
cd E:\project\payclip-workbuddy\backend
pip install -r requirements.txt
uvicorn app.main:app --host 0.0.0.0 --port 8000
```
- 启动后 **自动建表**（19 张表）并创建默认管理员。
- 健康检查：`http://localhost:8000/api/health` → `{"status":"ok"}`

### 3.3 启动前端

方式 A（开发模式，推荐）：
```bash
cd E:\project\payclip-workbuddy\frontend
npm install
npm run dev
# 开发服务器：http://localhost:5173 （Vite 已代理 /api → localhost:8000）
```

方式 B（直接托管已构建产物，无需 npm install）：
把 `frontend/dist/` 用任意静态服务器托管，或将 `dist/` 内容放到 Nginx 根目录即可。

### 3.4 登录

- 管理员账号：**`admin` / `admin123`**（拥有全部权限）
- 浏览器打开 `http://localhost:5173` 用该账号登录。

## 5. 推送到 GitHub（在本地执行）

沙箱无法直连 GitHub，需在**你的本地机器**上初始化仓库并推送：

```bash
cd E:\project\payclip-workbuddy
git init
git add .
git commit -m "init: payclip 薪资管理工具"
git remote add origin https://github.com/mountainhigh/payclip.git
git branch -M master
git push -u origin master
```

> 若 `mountainhigh/payclip` 仓库已存在且非空，先 `git pull --rebase origin master` 再 `git push`；
> 若提示需鉴权，配置 GitHub 个人访问令牌（PAT）或在推送时输入令牌作为密码。
