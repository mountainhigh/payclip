# 薪资管理工具 — 需求规格说明书 (SRS)

> 文档状态：需求确认版 v3（v2 基础 + 多租户 SaaS 改造 / H5 移动端 / 套餐定价 / 注册码机制，经 grill-me 逐层审查 6 大决策分支全部确认）
> 适用阶段：需求确定 → 整体设计 → 开发
> 技术栈：Python FastAPI + MySQL 8.0 + Vue3（PC: Element Plus / 移动端: Vant）+ JWT + bcrypt

---

## 1. 项目概述

### 1.1 背景与目标
面向 **10–20 人小型企业**（无自有 OA/IT 系统）的薪资管理工具，解决三类痛点：
- 多成分薪资（底薪 + 服务提成 + 销售提成 + 一次性业务提成 + 超额奖励）手工算易错；
- 收款登记、核对、内账校验流程分散；
- 客户/供应商/业务数据无统一台账，经营情况说不清。

### 1.1.1 产品形态升级（v3 新增 — SaaS 多租户）
本系统从单租户工具升级为 **PaaS/SaaS 多租户平台**，核心变更：
- **多租户隔离**：共享表 + `tenant_id` 方案，所有业务表加租户列，应用层强制过滤；
- **租户 = 薪资服务公司本身**（使用本 SaaS 的小微企业/财税工作室），其下管理自己的员工（User）和客户（Company）；
- **套餐定价**：1 元版（30 天全功能试用）→ 付费版（¥20/月 或 ¥199/年）；
- **注册码机制**：线下收款后发放注册码，用户凭码注册，暂不接入在线支付；
- **H5 移动端**：Vue3 + Vant 独立移动端，微信内分享 H5 链接引流，公众号菜单跳转；
- **移动端定位**：方便员工使用 + 业务引流，后续视验证结果再评估微信小程序。

### 1.2 产品形态（已确认）
- **PC Web 应用**：Vue3 + Element Plus，PC 优先；
- **H5 移动端**：Vue3 + Vant，独立移动端项目，微信内分享引流，公众号菜单跳转；
- **小程序**：Phase 2 评估，当前不做；
- 目标用户：主管/财务/老板用 PC，员工用 H5 移动端填报收款。

### 1.3 使用角色（三级角色体系，v3 新增）

**平台级角色：**

| 角色 | 说明 | tenant_id | 权限 |
|------|------|-----------|------|
| `super_admin` | 平台超级管理员（运营方） | NULL | 管理所有租户、生成注册码、手动开通付费、查看全局报表 |

**租户级角色：**

| 角色 | 说明 | tenant_id | 权限 |
|------|------|-----------|------|
| `tenant_admin` | 租户管理员（付费注册者） | NOT NULL | 本租户全功能 + 邀请员工 + 分配权限 |
| `employee` | 租户内普通员工 | NOT NULL | 按 `permissions` 数组限定 |

> 小公司一人多角色仍然支持，`tenant_admin` 可以同时持有所有 employee 权限。

## 2. 权限与角色模型

### 2.0 多租户隔离机制（v3 新增）
- **隔离方案**：共享表 + `tenant_id`，所有业务表加 `tenant_id` 列；
- **JWT 注入**：令牌 payload 新增 `tenant_id` 和 `role` 字段，`get_current_user` 返回租户上下文；
- **查询过滤**：所有业务查询自动带 `filter(tenant_id=current_tenant)`，防止跨租户泄漏；
- **super_admin** 的 `tenant_id=NULL`，可跨租户查询。

### 2.1 套餐与定价体系（v3 新增）

| 套餐 | 价格 | 功能 | 期限 |
|------|------|------|------|
| 试用版 | ¥1 | 全功能 | 30 天 |
| 月付版 | ¥20/月 | 全功能 | 30 天（可续） |
| 年付版 | ¥199/年 | 全功能 | 365 天（可续） |

**套餐状态机：**
```
注册（凭注册码）→ active（全功能 30 天）→ expired_readonly（只读降级 30 天）→ soft_deleted（软删除，可恢复）
                                          ↑                    │
                                          └─────续费───────────┘
```

**套餐规则：**
- 注册即必须付费（通过注册码机制，线下收款后发放）；
- 不拆免费功能，¥1 = 全功能 30 天；
- 到期后只读降级 30 天（数据可看不可改，写操作返回 403 + 续费引导）；
- 只读降级 30 天后软删除（数据保留，付费后可恢复）；
- 员工数 = 登录账号数（User 表行数），受套餐限制；
- 在线支付后置（当前由 `super_admin` 在后台手动标记已付费）。

### 2.2 权限点（功能权限）
- `payment:submit` — 填报收款
- `payment:verify` — 核对收款
- `salary:view` — 查看薪资/核对表
- `salary:manage` — 内账校验、触发提成/薪资计算
- `report:view` — 查看经营报表
- `admin:config` — 用户/客户/供应商/成本预设/奖励阶梯管理
- `tenant:admin` — 租户管理员（本租户全权限，v3 新增）

### 2.3 数据范围
- **SELF**：看自己的薪资 + **自己作为负责人的业务**对应的提成、报表数据。
- **ALL**：看全部。
- ❌ **TEAM 数据范围 Phase 1 直接去掉**（Team 表不启用），小公司一人多角色已足够，不留死字段。

### 2.4 租户入驻流程（v3 新增）
```
① super_admin 线下收款 → 在后台生成注册码（随机码，一次性使用）
② 用户在 H5 注册页输入：注册码 + 手机号 + 公司名称 + 密码
③ 系统自动创建：
    - Tenant 记录（name=公司名, plan='trial', status='active', trial_expires=now+30天）
    - User 记录（tenant_id=新租户, role='tenant_admin', permissions=['tenant:admin'] 全权限）
④ 注册码标记已使用
⑤ 用户登录，全功能 30 天
```

### 2.5 员工邀请机制（v3 新增）
- `tenant_admin` 在后台生成邀请链接（含 token）；
- 员工点开链接 → 填手机号 + 密码 → 自助注册加入该租户；
- 员工数受套餐限制（超限则拒绝注册，提示升级套餐）。

### 2.3 核对人分配机制（已确认）
- 员工**提交收款时选择核对人**；
- 有 `payment:verify` 权限的人，核对列表**只显示分配给自己**的收款（不受 SELF 限制，但受"被分配"限制）；
- 主管可单条通过/驳回，也可对自己名下的收款**批量通过**。

## 3. 功能需求

### 3.0 移动端功能（v3 新增 — H5 移动端）
基于 Vue3 + Vant 的独立 H5 移动端项目，微信内可分享：
- **登录**：手机号/用户名 + 密码登录；
- **工作台**：今日待办、本月收款概览、个人提成摘要；
- **收款填报**：选客户 → 关联业务 → 金额 → 渠道 → 截图上传 → 选核对人；
- **收款核对**：待核对列表、单条通过/驳回；
- **个人核对表**：查看自己某月的提成明细、薪资构成；
- **客户列表**：只读浏览客户信息；
- **我的**：个人信息、修改密码、退出登录。

### 3.1 客户管理
- 客户 CRUD：名称、区域标签（**扁平标签**，如 `["广州","佛山"]`，非层级）、是否新客、服务开始日期（用于销售提成 12 月计时）、备注。
- 客户下可同时存在**长期业务（订阅）**与**一次性业务**：
  - 长期业务：服务类型、账期（月/季/半年/年）、月费（**支持历史月费记录，按生效日期取值**）、是否成本类业务、月成本、供应商、服务负责人（→服务提成）、开发人（→销售提成）、开始日期、启停。
  - 一次性业务：项目类型、收入、成本、供应商、负责人（→一次性提成）、完成日期、是否收款、收款日期。
- **Excel 模板导入**历史客户与进行中业务（上线初始化用）。

### 3.2 供应商管理
- 供应商 CRUD：名称、类型（刻章/地址挂靠/审计/其他）、联系方式、备注。
- 供应商 = **成本提供方**，关联长期/一次性业务的成本。

### 3.3 收款管理（两种方式并存）
**A. 逐笔填报**（员工用，移动端适配）
- 选客户、**关联业务（必填）**、金额、日期、渠道（银行卡/微信/支付宝/对公/现金）；
- **截图必传**（单张 ≤5MB，JPG/PNG 格式，每条收款限传 3 张）；
- 选核对人；
- 标记公私用（私用 = 私人费用，详见 §4.8）。

**B. 批量导入流水**
- 采用**统一标准模板**（系统定义列名：日期/金额/渠道/备注/对方账户/公私标记），用户将银行/微信导出文件整理为该格式后导入；
- 导入时**先按备注关键字自动匹配客户**，匹配不成功的需**人工补充关联**（客户 + 业务）；
- 流水截图（OCR 识别）导入放 **Phase 2**；
- 导入后筛出公司交易，私人费用标记忽略（见 §4.8）。

**核对流程**：待核对 → 主管通过/驳回（填驳回原因）→ 标记公用/私用。

### 3.4 提成计算引擎（核心）
触发方式：**财务手动触发**（先完成内账校验，再点"计算当月"）。详见 §4。

### 3.5 薪资核算
- 应发 = 底薪 + 服务提成 + 销售提成 + 一次性业务提成 + 超额奖励(P2) + 补回 − 欠费扣款。
- 年终奖、个税社保 Phase 1 **只预留字段，不计算**（见 §4）。
- 生成月度薪资记录，员工可看自己，管理者看全部。

### 3.6 个人核对表
- 员工查看自己某月的：底薪、三类提成明细、欠费扣款、补回、合计；
- 提成明细可追溯（来源客户/业务）。

### 3.7 内账校验锁
- 财务点**"校验并计算当月"** → 先将当月数据**只读锁定**，再执行提成/薪资计算（**一个按钮两步走**）；
- **锁定后禁止新增/修改当月的收款记录**（前端拦截 + 后端校验）；
- **最近一个月的锁定可撤销**：撤销后可修改数据，改完后重新点击"校验并计算当月"；
- **历史月份（非最近一月）不可撤销锁定**，不可修改任何数据。

### 3.8 经营报表
- 按区域统计（基于客户区域标签聚合收款）；
- 按费用统计（长期业务成本 + 一次性业务成本，按类型分组）；
- 月度收款趋势（12 个月）；
- **Phase 1 支持 Excel 导出**。

### 3.9 系统设置
- 用户管理（增删改、权限、底薪、数据范围）；
- 成本预设管理（业务类型 → 默认成本，唯一约束，可改）；
- 超额奖励阶梯配置（Phase 1 仅维护配置，计算放 Phase 2）。

### 3.10 租户管理（v3 新增 — super_admin 专用）
- **注册码管理**：生成/查看/作废注册码；
- **租户列表**：查看所有租户、套餐状态、员工数、到期时间；
- **手动开通**：标记租户已付费、续费、调整套餐；
- **租户状态控制**：暂停/恢复/软删除/恢复。

### 3.11 数据导入/导出
- 导入：客户/业务 Excel 模板（上线初始化）；收款流水 CSV/Excel（Phase 1）。
- 导出：**薪资单（每人一页 sheet，含底薪/各项提成/扣款/补回/应发合计）+ 经营报表 Excel 导出（Phase 1 即做）**。

## 4. 业务规则详述（核心，已逐项确认）

### 4.1 三种提成类型

| 类型 | 计算基数 | 比例 | 计提时点 | 欠费规则 | 12月限制 |
|------|---------|------|---------|---------|---------|
| 服务提成 | 月费 | 15% | 服务当月 | 有（扣 5%） | 无 |
| 销售提成 | 收款金额 | 15% | 收款当月 | **无** | 有（12 月） |
| 一次性业务提成 | 收入−成本（毛利） | 20% | 业务完成月 | 有（扣 5%） | 无 |

- **一人可同时拿服务提成 + 销售提成**（同一客户，分别由服务负责人、开发人持有）。

### 4.2 欠费扣款（已确认）
- 基数 = **业务金额 × 5%**：
  - 服务提成：欠费扣 `月费 × 5%`；
  - 一次性业务：未收款扣 `毛利 × 5%`。
- 判定欠费：计费周期内该客户**无任何"已核对且公用"收款**即视为欠费；
- **账期展开规则（已确认）**：
  - 月付：每月独立判定，当月无已核对公用收款 → 欠费；
  - 季付/半年付/年付：**一次性付款覆盖整个账期对应月份**（如年付客户 1 月付了全年 → 1–12 月均不欠费）；若分期支付则按实际收款覆盖对应月份；
- **私用收款完全不参与欠费判定**（见 §4.8）。

### 4.3 补回规则（已确认）
- 触发：**收足才补回** —— 本月该客户收款金额 ≥ 当月应收金额时，补回之前月份因欠费扣的款；
- 补回金额 = **之前所有月份扣款总额**（多月扣款在收足当月一次性全部补回）；
- 补回计入**收足当月**的提成（正项）。
- **月中停用规则（已确认）**：长期业务月中停用，停用当月**按整月计提服务提成**（不按天折算）。

### 4.4 销售提成 12 个月窗口（已确认）
- 从 `company.service_start_date` 所在月起算，**含当月共 12 个自然月**（第 0–11 月可计提，第 12 月起停）；
- 例：service_start_date = 2025-03 → 窗口为 2025-03 至 2026-02（含首尾）；
- 仅对新客（`is_new_customer=true` 且有服务开始日）的开发人计算；
- 基于"收款当月"且"已核对 + 公用"的收款金额。
- **窗口期内业务启停不影响计时**（中途停止后恢复，12 个月窗口不中断、不延长）。

### 4.5 底薪
- 每位员工固定底薪，系统存一份，**每月自动带出**；可在用户管理修改。

### 4.6 超额阶梯奖励（Phase 1 预留，不计算）
- 配置规则示例：提成总额 >2 万额外奖 5%，>3 万部分按 10%；
- 算法待 Phase 2 确认（倾向"超额累进"，类似个税）；
- Phase 1 仅维护阶梯配置，**计算引擎不应用**。

### 4.7 年终奖 / 个税社保（Phase 1 预留字段）
- 年终奖：暂不计算，字段预留；
- 个税/社保/公积金：预留字段，后续接入；**本系统 Phase 1 只算"应发"**（底薪+提成+补回−扣款）。

### 4.8 私用 / 私人费用（已确认，重要）
- 定义：账务经理/老板导入银行或微信流水时，筛出的**个人日常交易**（非公司经营）。
- 处理：**完全不参与本项目的任何计算** —— 不计提成、不计欠费判定、不计入报表；
- 系统保留记录并标记"私用"，便于追溯，但所有计算与统计均排除。

### 4.9 离职员工提成（已确认）
- **离职即停**：离职次月起不再产生任何新提成（服务/销售/一次性均停）；
- 离职当月已计提的提成照常发放；
- 实现上：计算引擎仅对 `is_active=true` 员工生成当月提成。

## 6. 非功能需求
- 并发：小团队（<30 人）低频操作，MySQL 8.0 足够；**计算触发加简单状态锁（"计算中"标志位）防重复**；
- 安全：JWT 认证（含 tenant_id）、密码 bcrypt 哈希、上传文件类型/大小限制、XSS 转义、**多租户数据隔离**；
- 兼容性：Chrome/Edge 最新版；H5 移动端 **375px 起**适配（Vant 组件库）；
- 备份：MySQL 定时备份 + uploads 目录定期备份（部署阶段定）。
- **界面语言**：所有界面**强制中文**，包括日期选择器等组件。
- **删除约束**：
  - 客户/业务：有未收款或未结清时禁止删除；否则软删除（归档标记，数据保留）；
  - 员工：离职 = `is_active=false`，历史薪资/提成完整保留可查。

| G | 核对驳回后的处理 | **可修改重提**：驳回原因非必填但需 2 次确认避免误操作；无超时自动通过机制 | 收款流程 |
